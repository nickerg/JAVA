1.AWT
- Abstract Windows Toolkit(AWT)是最原始的Java GUI工具包。
- 优点：在Java技术的每个版本上都成为了一种标准配置，且非常稳定。不需要单独安装这个工具包，在任何一个Java运行环境中都可以使用
- AWT是一个具有有限GUI组件、布局管理器和事件的工具包（如窗体、文本标签、按钮、菜单等）。
- AWT要依赖于主机GUI的对等体控件来实现，其外观和行为在不同的主机上有所不同，具有局限性。

2.Swing
- Java Swing是Java Foundation Classes(JFC)的一部分，是试图解决AWT缺点的一个尝试
- Swing是在AWT组件基础上构建的。
- Swing使用了AWT的事件模型和支持类
- 将对主机控件的依赖性降至了最低，只为诸如窗口和框架之类的顶层组件使用对等体。
- Swing对基于对等体的组件使用的术语是重量级(heavyweight)，对模拟的组件使用的术语是轻量级(lightweight)。
- 基于MVC结构
- 可编程外观
- 呈现器与编辑器
- 支持以任意顺序构建GUI
- 可访问性
- 支持自动销毁
- 是标准Java的一部分

3.SWT
- 也是基于一个对等体实现。
- 克服了LCD的问题
- 定义了一组控件，可以用来构建大部分办公应用程序或开发者工具，然后可以按照逐个主机的原则，为特定主机所没有提供的控件创建模拟控件。
- 对于大部分现代主机来说，几乎所有的控件都是基于本地对等体的。这意味着基于SWt的GUI既具有主机外观，又具有主机的性能。
- 提供了扩充版本（通常使用"C"作为名字中的第一个字母）
- SWT应用程序实际上就是一个主机应用程序，会继承主机的优点和缺点
- 不能完全实现“一次编写，处处运行”，更像是“一次编写，处处测试”。
- 不支持GUI的自动销毁
- 仅支持自顶向下构建GUI
- 对Java2D的支持不够完善
- 不属于标准Java的一部分

4.只有在一种场合不适用于Swing：在使用SWT开发的程序上继续编写界面。
